核心概念

工作区 你正在写的文件（VS Code 里看到的）
→
暂存区 放进篮子 git add
→
仓库 正式存档 git commit

日常使用（每次都做这三步）

git add .
git commit -m "这改了什么"
git push

全套命令速查

命令 什么时候用 频率
git init 新项目初始化 一次
git status 看看现在什么状态 经常
git add . 放进篮子 每次提交前
git commit -m "..." 正式存档 做完一个功能
git push 推送到 GitHub Commit 之后
git log 看历史记录 偶尔
git reset 把篮子倒空 加错了时

新项目首次配置（此项目已完成，以后新项目才需要）

git init
echo "node_modules/" > .gitignore
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
git remote add origin git@github.com:用户名/仓库名.git
git branch -M main
git push -u origin main

常见坑

1. Git Bash 里波浪号用英文输入法打（Shift + Tab上面那个键），中文的 ～ 不行
2. 命令手打，不要从网页复制（会带特殊字符报错）
3. commit 之后 git status 看到 nothing to commit 才是正常的
4. 红色文件名 = Git 不认识它；绿色 = 在篮子里准备存
5. 文件没保存（Ctrl+S）的话 git add 不会包含最新改动
